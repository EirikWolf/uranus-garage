# Teknisk Revisjon — 2026-03-29

## Fase 1: Kritiske feil fikset

### Sikkerhet

1. **Mass assignment i Listing PATCH** — `src/app/api/listings/[id]/route.ts`
   Uvalidert request body ble sendt direkte til Prisma update. Lagt til `updateListingSchema` med Zod-validering.

2. **Privat fork-lekkasje** — `src/app/api/forks/route.ts`
   Enhver bruker kunne se andres private forks via `userId`-parameter. Lagt til sjekk: kun eier ser private forks.

3. **RAPT POST uten auth** — `src/app/api/rapt/sync/route.ts`
   POST-endepunktet for manuelle malinger krevde ingen autentisering. Lagt til `auth()`-sjekk.

4. **E-postlekkasje** — `src/app/api/brew-swap/[id]/route.ts`, `src/app/api/listings/[id]/route.ts`
   Brukernes e-post ble eksponert i API-svar. Fjernet `email` fra select-queries.

5. **z.any() validering** — `src/lib/validations.ts`
   Grains, hops, yeast, additions og process brukte `z.any()`. Erstattet med typede Zod-skjemaer.

### Bugs

6. **scaleAdditions gjorde ingenting** — `src/lib/scaling.ts`
   Funksjonen returnerte uendrede data. Implementert skalering av numeriske verdier i string-amounts.

7. **Lineage uten dybdegrense** — `src/app/api/forks/[id]/lineage/route.ts`
   Rekursiv traversering uten stoppkriterium. Lagt til MAX_TREE_DEPTH=50 og maks 500 noder.

8. **env.ts krasjet ikke** — `src/lib/env.ts`
   Manglende env-variabler logget bare en advarsel. Kaster na Error i production.

## Fase 2: Alle utestående forbedringer implementert

### 1. Next.js Proxy for global auth + CSP headers — `src/proxy.ts`
- Proxy (Next.js 16 erstatter middleware) som:
  - Blokkerer uautentiserte POST/PATCH/DELETE til beskyttede API-routes tidlig (cookie-sjekk)
  - Setter CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy pa alle responses
  - Tillater offentlige GET-endepunkter og auth-routes gjennom

### 2. Lineage migrert til rekursiv CTE — `src/app/api/forks/[id]/lineage/route.ts`
- Erstattet N+1 rekursive queries med en enkelt SQL CTE
- Henter ancestors + descendants + ratings + child counts i ett kall
- Beholder MAX_DEPTH=50 og MAX_NODES=500 begrensninger

### 3. Sanity revalidering — `sanity/lib/client.ts`, `src/lib/sanity.ts`
- Eksporterer `SANITY_REVALIDATE = 60` fra client
- Alle 14 GROQ-queries bruker na `{ next: { revalidate: 60 } }`
- Innhold revalideres hvert 60. sekund via ISR

### 4. Prisma _avg for ratings — `src/app/api/forks/route.ts`, `src/app/api/forks/[id]/route.ts`
- Fork-liste: Bruker `rating.groupBy` med `_avg` for batch-beregning
- Fork-detalj: Bruker `rating.aggregate` med `_avg`
- Eliminerer lasting av alle individuelle ratings til JS

### 5. Error boundaries og loading states
- `src/app/error.tsx` — Global error boundary med retry-knapp
- `src/app/not-found.tsx` — 404-side med lenke tilbake
- `src/app/loading.tsx` — Spinner-komponent for alle routes
